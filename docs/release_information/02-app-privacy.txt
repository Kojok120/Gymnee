==========================================================
App のプライバシー（栄養ラベル）回答ガイド
App Store Connect → App のプライバシー で設定
==========================================================
※ アプリの PrivacyInfo.xcprivacy と、この回答と、プライバシーポリシーの3つを必ず一致させる。
※ 下記は現実装（Supabase 同期・本人に紐づく保存・トラッキング無し）に基づく推奨回答。

【データ収集の有無】
「このAppはデータを収集しますか？」→ はい

----------------------------------------------------------
収集するデータ種別（すべて「ユーザーに紐づく / トラッキングには使用しない / 目的=アプリの機能」）
----------------------------------------------------------
■ 健康とフィットネス
  - 健康 / フィットネス（体重・体組成・ワークアウト）
  - 用途: アプリの機能 / 紐づけ: あり / トラッキング: なし

■ 写真またはビデオ
  - 用途: アプリの機能 / 紐づけ: あり / トラッキング: なし

■ 位置情報
  - 正確な位置情報（近隣ジム補完・自動チェックイン）
  - 用途: アプリの機能 / 紐づけ: あり / トラッキング: なし

■ 連絡先情報
  - 名前（表示名）
  - メールアドレス（Sign in with Apple / アカウント）
  - 用途: アプリの機能, アカウント管理 / 紐づけ: あり / トラッキング: なし

■ ユーザーコンテンツ
  - その他のユーザーコンテンツ（メモ・投稿・共有カード等）
  - 用途: アプリの機能 / 紐づけ: あり / トラッキング: なし

■ 識別子
  - ユーザーID（Supabase アカウント識別子）
  - デバイスID（プッシュ通知トークン）※プッシュ送信を有効化した場合
  - 用途: アプリの機能 / 紐づけ: あり / トラッキング: なし

----------------------------------------------------------
トラッキング
----------------------------------------------------------
「データをトラッキングに使用しますか？」→ いいえ
（他社サービス横断の広告トラッキングは行わない。App Tracking Transparency 不要）

----------------------------------------------------------
注意（重要）
----------------------------------------------------------
・HealthKit データは広告・マーケティングに利用しない／第三者へ販売しない（ガイドライン 5.1.3）。
  → プライバシーポリシー第3項に明記済み。
・アフィリエイト送客は SFSafariViewController での外部遷移であり、アプリ内トラッキングではない。
・プッシュ送信機能を未実装でリリースする場合、デバイスIDの収集は外してよい（実装時に追加）。
