# Gymnee「記録」リデザイン 仕様書 v1

> 既存の記録フロー（`WorkoutLoggerView` / `SetRowView` / `WorkoutHomeView` を中心とした「種目選択→weight/reps数値入力→完了チェック」方式）を全面的に置き換える。
> 既存コードの再利用は前提にしない（ベスト仕様を優先し、不要なものは削除して作り直す）。
> DEV段階のため既存データの全消去は許容（スキーマ移行は不要・`makeContainer` の作り直しで対応）。

## 1. コンセプト

入力（数値タイピング）を廃し、**タップだけで記録**する。種目は「重量候補3／種目名／reps候補3」の**カード**で表現し、weight と reps をタップした回数がセット記録になる。候補は使うほど各自に最適化される。記録ハブ（`WorkoutHomeView`）は廃止し、「記録」タブの実体をこのタップ・ロガーにする。

## 2. 記録画面の構成（上から）

- **① モードプルダウン（選択シート）**：`フリー / 計画(今日あれば) / ルーティン各種`。デフォルト＝今日計画あれば「計画」、無ければ「フリー」。
  - ①は単なるメニューではなく**選択シート**。各ルーティン行に「編集」、末尾に「＋ルーティンを作る」。
  - ①は**カードの絞り込み**。記録（②）は1ワークアウトに蓄積され、モードを切り替えてもログは消えない。
  - 計画/ルーティンを選ぶと、そのワークアウトはそれに紐づく（達成率・カレンダー表示に反映）。フリーは紐づきなし。足りない種目はフリーに切替えて追加（ルーティン＋おまけ運用）。
- **② 記録ログ**：タップ順・**時系列**で積層（各行に時刻）。**行スワイプで削除／行タップでミニ編集**（weight・reps変更）。済みセット数・履歴は“ここ”で見る（カードには重ねない）。
- **③ カテゴリ（フリー時のみ）**：部位を選ぶと、その部位へ**スクロール・ジャンプ**（フィルタではない）。
- **④ 種目カード群**（下記 §3）。
- **⑤ フッター**：左＝レストカウントダウン（既定90秒・設定で変更）／右＝ワークアウト経過時間。

## 3. カード（ミニマル：補助情報は一切載せない）

カードは「**weightスロット3／種目名／repsスロット3**」のみ。済みセット数・前回サマリー・前回重量ラベルは出さない。
3つの計測タイプを `Exercise.measurementType` で保持する：

| タイプ | 上スロット | 中 | 下スロット | 記録 |
|---|---|---|---|---|
| weight | 重量3（昇順） | 種目名（＋片側/両側ラベル） | reps3 | weight × reps |
| bodyweight（自重） | 加重3（+0 / +5 / +10 …） | 種目名 | reps3 | 加重 × reps |
| time（時間） | 秒3（45 / 60 / 90 …） | 種目名 | （なし） | duration 秒 |

**タップ操作**
- weight（加重/秒）をタップして**固定（アーム）** → reps をタップで「固定値 × その reps」を1セット記録 → ② に即追加・⑤レスト発火。
- 別の weight をタップで持ち替え（**ピラミッド 40→65→50 を編集ゼロ**で記録）。
- 範囲外の値は**スロット長押し → キーパッド**で入力（次回からスロットに学習）。

## 4. 候補スロットの出どころ（優先順位）

- **重量（/加重/秒）**
  1. ルーティン/計画の登録トリオ（事前に入れた値。**初記録でも数値が出る**）
  2. 履歴：前回その種目で使った重量（distinct を昇順、4種類以上なら使用頻度の高い3つ）
  3. 空スロット ＋ 初回オンボード説明（**フリーの初見種目のみ**）
- **reps**：よく使う3rep（だいたい安定、例 8/10/12）。ルーティンは登録値。
- **進化**：ワークアウト完了時に「**『胸の日』の目標を 40/55/65 に更新する？ [更新][そのまま]**」を確認。`RoutineExercise` ごとに独立保持＝同種目が複数ルーティンで別値でも衝突しない。

## 5. セッション・ライフサイクル

記録タブは**待機表示**（カードは見えるがタイマー `--:--`）→ **最初のセットをタップで自動開始**＋経過タイマー始動 → **「完了」→ サマリー**（種目数 / セット数 / 総ボリューム / 所要時間 / PR、共有・分析・閉じる）。空ワークアウトは作られない。
**PR検出はタップ毎、フィード発行は完了時**（現行踏襲）。

## 6. ルーティン

「**3重量(or 加重/秒)＋3reps＋種目（順序）**」を保持。作成/編集は①の選択シートから（＋ルーティンを作る／行の編集）。**事前に重量を入力でき、初記録でもその値が出る**。入力欄は計測タイプに追従。

## 7. 計画（AIバッチ）

カレンダーから「計画を立てる」。AIが**期間分**を一括設計。
- 入力：**連携カレンダー予定 ＋ 部位ごとの回復値（`RecoveryAnalyzer`）＋ ルーティン ＋ 種目 ＋ 週目標**。
- 中身：**ハイブリッド（ルーティン優先 ＋ AI補完）**。
- 生成後：**日の移動／削除／再生成**（種目細部はルーティン編集側で）。
- 連携は **Apple のみ**（Google は将来）。
- 各日の計画が記録①の「計画」モードになる。

## 8. 種目

作成＝フリーの「＋種目」、編集＝**カード長押し**（名前・部位・器具・**計測タイプ**・**片側/両側**・削除）。片側/両側の既定は器具で自動（ダンベル/ケトルベル＝片側、他＝両側）。

## 9. カレンダー／ナビ変更

- 「最近のワークアウト」廃止（履歴はカレンダー日タップ → ワークアウト詳細）。
- 計画・ルーティン管理・種目管理は記録画面とカレンダーに分散（使う場所で作る）。「同じ内容で開始」はルーティン化で代替。

## 10. データモデル変更（作り直し前提）

| エンティティ | 変更 |
|---|---|
| `Exercise` | **＋ `measurementType`**（weight / bodyweight / time）。`weightMode`（両/片）は維持・器具で既定 |
| `ExerciseSet` | **＋ `durationSeconds: Int?`**。`reps`/`weight` 維持。**廃止**：`rpe` / `rir` / `type(SetType)` / `weightModeOverride`。`isCompleted` は実質常に true。`createdAt` は②の時系列順に使用（維持） |
| `RoutineExercise` | **＋ `weightSlots: [Double]` / `repSlots: [Int]`**（時間種目は `durationSlots: [Int]`）。`targetReps` は `repSlots` に吸収。`targetSets`/`restSeconds` は計画用に残すが記録UIでは非表示 |
| `WorkoutExercise` | **廃止**：`supersetGroup`。`restSeconds` は使わない |
| `PlannedWorkout` | `detailJSON` に slot / ルーティン参照を反映（ハイブリッド対応） |
| 設定 | **＋ `gymnee.restSeconds`（既定90）**、**＋ `gymnee.recordOnboardingShown`** |

## 11. 既定の決め（明示的に投票していない箇所。違えば指摘で覆す）

- reps スロットは学習するが基本安定（8/10/12 等）。
- `targetSets` はカード非表示（セット数は自由・②で見える）。計画AIの入力としては残す。
- 完了サマリー・PR・シェアは現行を踏襲。
- 種目別メモ（`WorkoutExercise.note`）は高速動線から外す（残すなら詳細側）。

## 12. 廃止/簡素化される既存要素

- `SetRowView` / `NumberField`（数値直接入力）
- `WorkoutHomeView`（記録ハブ：最近のワークアウト・各種導線）
- `WarmupCalculator`（ウォームアップ自動生成）
- SetType（warmup/drop/superset）、RPE/RIR、per-set weightMode override、superset グループ
- 種目別レスト（`WorkoutExercise.restSeconds`）→ アプリ全体の単一デフォルトへ
